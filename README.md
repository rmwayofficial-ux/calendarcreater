# 予約状況カレンダー作成アプリ

セラピストさん向け。**午前・午後のマスをタップして ◎○△× を切り替えるだけ**で、
月間の予約状況カレンダーを作り、**JPEG画像でダウンロード**できます（ブログ貼り付け用・横長）。

- サーバー不要。すべてブラウザ内で完結し、入力内容は端末のローカルにのみ保存されます。
- **日曜始まり**のカレンダー。祝日（振替休日・国民の休日含む / 1980〜2099年）を自動表示。
- カレンダー周りの余白に、**その月の季節に合った背景イラスト**（雪/梅/桜/春/新緑/梅雨/七夕/夏/月見/紅葉/銀杏/冬）を自動で添えます。

## マークの色と意味

記号は色付き・太字で表示されます（背景の塗りなし）。

| マーク | 色 | 意味 |
|:---:|:---:|---|
| ◎ | 赤 | 空き十分 |
| ○ | 緑 | 空きあり |
| △ | 青 | 残りわずか |
| × | 濃グレー | 満席・受付終了 |
| − | グレー | 受付不可（予約を入れられない日。マーク空白時に表示） |

カレンダー本体の配色は、白（メイン）／teal `#4ab4aa`（アクセント：見出し・曜日帯）／グレー `#808080`（枠・罫線）。

## 使い方

```bash
npm install      # 初回のみ
npm run dev      # 開発サーバー起動（http://localhost:5173）
npm run build    # 本番ビルド（dist/ に出力 → 静的ホスティングにアップ可）
```

### 操作
1. 年月を選ぶ（◀▶ または プルダウン）
2. タイトルを入力（既定は「○月の予約状況」。見出しは大きく表示されます）
3. 各日の「AM」「PM」のマスをクリックすると `空白 → ◎ → ○ → △ → ×` と切り替わる（空白は横棒「−」＝受付不可で表示）
4. プレビュー上の「サイズ」で投稿先（ブログ用／Facebook用／Instagram用）を選ぶ
5. プレビューを確認して **「📥 JPEGをダウンロード」**
6. ダウンロードした画像をブログ・SNSにアップ

### 画像サイズ（投稿先フォーマット）
プレビュー上の「サイズ」ボタンで切り替えます。選んだサイズは自動保存され、ファイル名にも付きます（例：`予約状況_2026-06_Instagram.jpg`）。

| サイズ | 縦横比 | 用途 |
|---|---|---|
| ブログ用 | 内容に合わせた標準サイズ | 記事への貼り付け |
| Facebook用 | 1:1（正方形） | Facebookフィード |
| Instagram用 | 4:5（縦長） | Instagramフィード |

カレンダー本体は同じレイアウトのまま、周囲の余白（季節背景）で各サイズに合わせます。

### ことばで入力（特別な日）
各日のマスの下に**自由入力欄**があります。特別なイベントや出張など、◎○△× では表せない日は、ここにことばを入力します。
- **ことばだけ入力（マークは空白）** → その日はことばが大きく中央に表示されます（例：「大阪へ出張のため終日お休み」）
- **マーク＋ことば** → ◎○△× の下に小さくことばが表示されます（例：AM○・PM× ＋「夕方ヨガイベント」）
- 長い文は自動で折り返し、入りきらない分は「…」で省略されます

### デザイン切り替え
プレビューのすぐ上にある小さな「デザイン」ボタンで **やわらか／クール／シャープ／あたたか** の4種類を選べます（出力JPEGの色味・角の丸み・枠線・背景の質感が変わります。記号の色は共通）。選んだデザインは自動保存されます。

- **クール** は背景が季節の色に変わるだけで、季節イラストは付きません（すっきりした見た目）。
- **シャープ** は背景色が変わらない無地でクリアな印象。デザインは固定で、毎月「月日」だけが変わります。

### 使い方ページ・ヘルプページ
ヘッダー（またはフッター）のメニューから **「使い方」「ヘルプ」** を開けます。使い方は操作の流れ、ヘルプはよくある質問（データの保存場所・別端末への移行・トラブル時の確認など）をまとめています。

> スマホ・小さい画面では、上部の「✏️ 編集 / 👁 プレビュー」タブで表示を切り替えられます（編集とプレビューを行き来する上下スクロールが不要になります）。初めて開いたときは、使い方の案内が表示されます。

### ファイルで保存・読み込み（バックアップ／別端末への移行）
- **💾 ファイルに保存**：保存済みの全月データを1つの `.json` にダウンロード
- **📂 ファイルから読み込み**：その `.json` を読み込んで復元
- 別の端末・ブラウザにデータを移したいときや、バックアップ・共有に使えます

### 便利機能
- **前月をコピー**：前の月の入力内容を引き継ぎ、変更点だけ直す
- **曜日まとめ設定**：「毎週日曜は空白（定休）」などをワンタッチ
- **全部クリア**：その月のマークを一括消去
- **自動保存**：入力は自動でブラウザに保存され、次回開くと続きから（端末ごとに独立）

## 技術構成
- React 18 + TypeScript + Vite
- 画像生成：HTML Canvas に直接描画 → `toDataURL('image/jpeg')`
- 保存：localStorage（月ごと）

### ファイル早見表（`src/`）
| ファイル | 役割 |
|---|---|
| `App.tsx` | 画面（編集UI・操作・状態管理） |
| `renderCanvas.ts` | 出力JPEGの描画（レイアウト・色・テーマ適用） |
| `theme.ts` | デザインテーマ4種（やわらか/クール/シャープ/あたたか） |
| `Guide.tsx` / `Help.tsx` | 使い方ページ・ヘルプページ |
| `season.ts` | 月ごとの季節イラスト（背景モチーフ） |
| `calendar.ts` | 日曜始まりのカレンダー生成 |
| `holidays.ts` | 日本の祝日計算 |
| `storage.ts` | localStorage保存・テーマ・ファイル保存/読み込み |
| `types.ts` | マーク（◎○△×）とデータ型 |
| `styles.css` | 操作画面の見た目 |

### よくある調整ポイント
- マークの色：`renderCanvas.ts` の `MARK_INK`（編集画面側は `styles.css` の `--double` 等）
- テーマの色・枠線・季節イラストの寄せ具合：`theme.ts`
- 季節モチーフの絵柄：`season.ts`
- レイアウト寸法（行高・余白・見出しサイズ）：`renderCanvas.ts` 上部の定数
