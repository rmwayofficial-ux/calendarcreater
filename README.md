# 予約状況カレンダー作成アプリ

セラピストさん向け。**午前・午後のマスをタップして ◎○△× を切り替えるだけ**で、
月間の予約状況カレンダーを作り、**JPEG画像でダウンロード**できます（ブログ貼り付け用・横長）。

- サーバー不要。すべてブラウザ内で完結し、入力内容は端末のローカルにのみ保存されます。
- **日曜始まり**のカレンダー。祝日（振替休日・国民の休日含む / 1980〜2099年）を自動表示。
- カレンダー周りの余白に、**その月の季節に合った背景イラスト**（雪/梅/桜/春/新緑/梅雨/七夕/夏/月見/紅葉/銀杏/冬）を自動で添えます。

## マークの色と意味

記号は色付き・太字で表示されます（背景の塗りなし）。

| マーク | 色 | 意味 |
|:---:|:---:|---|
| ◎ | 赤 | 空き十分 |
| ○ | 緑 | 空きあり |
| △ | 青 | 残りわずか |
| × | 濃グレー | 満席・受付終了 |
| − | グレー | 受付不可（予約を入れられない日。マーク空白時に表示） |

カレンダー本体の配色は、白（メイン）／teal `#4ab4aa`（アクセント：見出し・曜日帯）／グレー `#808080`（枠・罫線）。

## 使い方

```bash
npm install      # 初回のみ
npm run dev      # 開発サーバー起動（http://localhost:5173）
npm run build    # 本番ビルド（dist/ に出力 → 静的ホスティングにアップ可）
```

### 操作
1. 年月を選ぶ（◀▶ または プルダウン）
2. タイトルを入力（既定は「○月の予約状況」。見出しは大きく表示されます）
3. 各日の「AM」「PM」のマスをクリックすると `空白 → ◎ → ○ → △ → ×` と切り替わる（空白は横棒「−」＝受付不可で表示）
4. プレビュー上の「サイズ」で投稿先（ブログ用／Facebook用／Instagram用）を選ぶ
5. プレビューを確認して **「📥 JPEGをダウンロード」**
6. ダウンロードした画像をブログ・SNSにアップ

### 画像サイズ（投稿先フォーマット）
プレビュー上の「サイズ」ボタンで切り替えます。選んだサイズは自動保存され、ファイル名にも付きます（例：`予約状況_2026-06_Instagram.jpg`）。

| サイズ | 縦横比 | 用途 |
|---|---|---|
| ブログ用 | 内容に合わせた標準サイズ | 記事への貼り付け |
| Facebook用 | 1:1（正方形） | Facebookフィード |
| Instagram用 | 4:5（縦長） | Instagramフィード |

カレンダー本体は同じレイアウトのまま、周囲の余白（季節背景）で各サイズに合わせます。

### ことばで入力（特別な日）
各日のマスの下に**自由入力欄**があります。特別なイベントや出張など、◎○△× では表せない日は、ここにことばを入力します。
- **ことばだけ入力（マークは空白）** → その日はことばが大きく中央に表示されます（例：「大阪へ出張のため終日お休み」）
- **マーク＋ことば** → ◎○△× の下に小さくことばが表示されます（例：AM○・PM× ＋「夕方ヨガイベント」）
- 長い文は自動で折り返し、入りきらない分は「…」で省略されます

### デザイン切り替え
プレビューのすぐ上にある小さな「デザイン」ボタンで **やわらか／クール／シャープ／あたたか／ビタミン** の5種類を選べます（出力JPEGの色味・角の丸み・枠線・背景の質感が変わります。記号の色は共通）。選んだデザインは自動保存されます。

- **やわらか** — ミントクリーム背景＋季節モチーフあり。若葉のフレッシュ感を意識した上品な仕上がり。
- **クール** — 淡いクールミスト背景＋季節イラストなし。すっきりモダン。
- **シャープ** — 明るめネイビー背景＋白カードが浮かぶスタイリッシュな仕上がり。背景は毎月同じ。
- **あたたか** — 温かみのある生成り背景＋淡い季節色のティント＋季節モチーフ。**唯一、月ごとに背景色が変化するテーマ**。
- **ビタミン** — 蛍光レモン背景＋ホットピンクの太い枠で、原宿系の元気なポップ仕上がり。

### 使い方ページ・ヘルプページ
ヘッダー（またはフッター）のメニューから **「使い方」「ヘルプ」** を開けます。使い方は操作の流れ、ヘルプはよくある質問（データの保存場所・別端末への移行・トラブル時の確認など）をまとめています。

> スマホ・小さい画面では、上部の「✏️ 編集 / 👁 プレビュー」タブで表示を切り替えられます（編集とプレビューを行き来する上下スクロールが不要になります）。初めて開いたときは、使い方の案内が表示されます。

### ファイルで保存・読み込み（バックアップ／別端末への移行）
- **💾 ファイルに保存**：保存済みの全月データを1つの `.json` にダウンロード
- **📂 ファイルから読み込み**：その `.json` を読み込んで復元
- 別の端末・ブラウザにデータを移したいときや、バックアップ・共有に使えます

### 便利機能
- **前月をコピー**：前の月の入力内容を引き継ぎ、変更点だけ直す
- **曜日まとめ設定**：「毎週日曜は空白（定休）」などをワンタッチ
- **全部クリア**：その月のマークを一括消去
- **自動保存**：入力は自動でブラウザに保存され、次回開くと続きから（端末ごとに独立）

## 技術構成
- React 18 + TypeScript + Vite
- 画像生成：HTML Canvas に直接描画 → `toDataURL('image/jpeg')`
- 保存：localStorage（月ごと）

### よくある調整ポイント
- マークの色：`renderCanvas.ts` の `MARK_INK`（編集画面側は `styles.css` の `--double` 等）
- テーマの色・罫線（経線）の太さ/色・季節あしらいの寄せ具合・背景の挙動（`seasonTint: true` で月ごと変化／`false` なら `plainBg` 固定）：`theme.ts`
- 季節モチーフの絵柄：`season.ts`（`drawSeasonMotif`）／配置・濃さ：`renderCanvas.ts` の `drawDecorations`
- レイアウト寸法（行高・余白・見出しサイズ）：`renderCanvas.ts` 上部の定数
- 出力サイズ（ブログ/Facebook/Instagram の縦横比）：`renderCanvas.ts` の `FORMATS`
- 年の選択範囲（今年-1〜+5年・自動前進）：`App.tsx` の `yearOptions`
- 見出し（年・タイトル・区切り線）：`renderCanvas.ts` の見出しブロック

### 実装済みの主な機能（場所）
- 初回お知らせモーダル「ブラウザで開く／ホーム画面に追加」：`App.tsx`（`showInstall` 状態＋ `detectPlatform`/`detectInApp`/`isStandalone` ヘルパー）。閉じると `storage.ts` の `installNoticeSeen` で以後非表示。iOS/Android/PC で手順を出し分け、アプリ内ブラウザ（LINE/Instagram等）検出時は警告文を強調。CSSは `.install-overlay` / `.install-modal` ほか。
- 初回ガイド／使い方・ヘルプページ：`App.tsx`（`page` 状態）＋ `Guide.tsx` / `Help.tsx`
- 出力デザイン5テーマ：`renderCanvas.ts` ＋ `theme.ts`（やわらか／クール／シャープ／あたたか／ビタミン）＋ `season.ts`。あたたかのみ `seasonTint: true` で月ごとに背景の表情が変化、他4テーマは固定背景（`plainBg`）。
- 出力サイズ切替（自動保存・ファイル名に付与）：`FORMATS` ＋ `storage.ts`（`loadFormatId`/`saveFormatId`）
- スマホ最適化（編集/プレビュー タブ・サイズ/デザインを一列・補助操作の並び替え）：`styles.css` のメディアクエリ＋`App.tsx`（`mobileView`）
- 終了時の未保存ガード（`beforeunload`）＋保存ボタン強調：`App.tsx`（`unsavedToFile`）
- マーク：空白は横棒「−」＝受付不可（`renderCanvas.ts` の `drawSlot` ／ `types.ts` の `MARK_LABEL`）

### 出力画像の見た目を手元で確認する方法（ブラウザ不要）
ローカルに `@napi-rs/canvas`（ネイティブ依存なしのprebuilt）を一時導入すると、Node から `drawCalendar` を呼んでPNGを書き出せる。
```bash
npm i --no-save @napi-rs/canvas
# 1) ./node_modules/.bin/esbuild で renderCanvas を Node 向けに bundle（entry で drawCalendar/THEMES を re-export）
# 2) createCanvas(10,10) を drawCalendar(canvas, data, theme, formatId) に渡し canvas.toBuffer('image/png') を保存
```
注意：この方法は日本語フォントと ◎○△ の字形が無い環境だと文字が□になる（レイアウト・配色・罫線・余白の確認用）。文字や記号の最終確認は実ブラウザ（`npm run dev` の右側プレビュー）で行う。
